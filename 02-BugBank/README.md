# BugBank - Casos de Teste Manuais

## Sobre o projeto

Este projeto foi desenvolvido como parte do meu processo de aprendizado em Quality Assurance e integra meu portfólio de testes manuais. Seu objetivo é documentar casos de teste para a funcionalidade de cadastro de usuários do BugBank.

Os cenários contemplam testes positivos e negativos, abordando validações comuns encontradas em sistemas de cadastro.

---

## Sistema testado

- **Sistema:** BugBank
- **Funcionalidade:** Cadastro de Usuário
- **Tipo de teste:** Testes Manuais
- **Versão:** Atual (ambiente disponível durante a execução)

---

## Objetivos

- Validar o fluxo de cadastro de usuários.
- Verificar validações de campos obrigatórios.
- Testar cenários positivos e negativos.
- Exercitar documentação de testes utilizando boas práticas.

---

## Casos de teste

- CT-001 – Cadastro com e-mail válido
- CT-002 – Cadastro com e-mail já existente
- CT-003 – Cadastro com senhas diferentes
- CT-004 – Cadastro com e-mail em formato inválido
- CT-005 – Cadastro sem informar o campo Nome
